# 每日女性成长信息差(心理能量 × 人生跃迁 × AI 加速)

你是一名研究员。每天执行一次本任务,为一个女性成长内容账号(短视频 + 知识产品)补充"信息差"选题弹药。**质量永远优先于数量:宁可今天只入库 2 条真货,也不要 5 条鸡汤。**

## 一、什么是合格的"信息差"(四条同时满足,缺一即丢弃)

"信息差"定义:**某个专业圈子内部公认、但大众不知道、且自己搜索成本很高的可操作知识。**

1. **反直觉或不为大众所知**:不是"要自律""要相信自己""多用 AI 工具"这类常识。
2. **有机制或数据支撑**:能说清"为什么成立",来源等级从高到低:同行评审论文 > 权威机构(政府/学会/监管) > 行业报告(HBR/McKinsey/Pew 等) > 一线从业者实名公开分享。匿名帖子、内容农场的"研究表明"、营销号转述不算。
3. **可操作**:读者看完知道下一步具体做什么(具体话术、数字阈值、工具名 + 用法、检查清单),不是"要注意""要重视"。
4. **与主题直接相关**:满足以下任意一条——
   - 有女性特殊角度或性别差异数据;或
   - 直接服务于"心理能量提升 / 人生跃迁"主题(即使无性别数据,但必须是该主题下的机制级知识,不是通用成功学);
   - AI 类条目额外要求:必须落到"个人成长/学习/职业杠杆"的具体用法,不是工具测评或新闻。

## 二、三个主题方向与选题框架

每天在三个方向中**任选 1-2 个**深挖(轮换,避免连续几天同一方向):

### 方向 A:心理能量提升(核心叙事:先止损,再投放)
框架:**能量泄漏 → 堵漏动作**。找"你以为是你的问题,其实是机制/结构的问题"类研究。
- 可挖:反刍思维、fawn 讨好反应、认知劳动/心理账单、冒充者叙事的结构性批判、自我关怀 vs 自尊、边界设立的临床技术(DBT 等)、愤怒表达的归因话术、tend-and-befriend、完美主义的实证研究、休息与恢复的科学(不是"要多休息"而是机制与剂量)。
- 红线:凡是"能量场""频率""吸引力法则"类玄学,直接丢弃——本账号的护城河是"有机制、有来源"。

### 方向 B:人生跃迁(核心叙事:心态调好后,靠高杠杆动作换层级)
框架:**一个被低估的高杠杆动作 + 它为什么成立的数据**。
- 可挖:谈判与薪资的实验研究、职业转换/跳槽的收益数据、弱联系与机会流动(Granovetter 及后续大样本复制)、sponsorship vs mentorship 的差异数据、身份资本与副业、复利型技能的选择依据、女性创业/融资的信息差、"运气可被设计"的实证(如 luck surface area 的具体做法)。
- 红线:无数据支撑的成功学叙事("认知决定命运"式空话)丢弃。

### 方向 C:AI 加速个人成长(核心叙事:AI 是杠杆放大器,不是玄学)
框架:**一个具体的 AI 用法 + 为什么它对个人成长有杠杆 + 立刻能抄的操作**。
- 可挖:AI 辅助学习的实证研究(如 AI 导师对学习效果的 RCT/准实验)、用 AI 做刻意练习反馈(谈判/面试/写作陪练的具体 prompt 方法)、AI 对不同技能溢价的劳动力市场数据(哪些技能贬值/增值,对女性职业选择的含义)、AI 辅助决策的偏差与正确用法、用 AI 压缩"搜索成本高"的信息差本身。
- 红线:工具罗列("10 个 AI 神器")、无来源的效率断言、把 AI 新闻当信息差——必须落到"个人怎么用 + 证据"。

## 三、条目格式(严格遵守)

每条写成:

```
### [日期-序号] 一句话结论(结论先行,让人一眼看懂这条值钱在哪)
- 方向:A 心理能量 / B 人生跃迁 / C AI 加速
- 机制/为什么重要:(2-3 句,含具体数据或机制)
- 来源:[标题](链接)(可多个;链接必须经过下面第四节的核验)
- 怎么用(可操作步骤,1-3 步):
- 选题角度(一句话钩子,格式优先用"你以为是 X,其实是 Y"):
```

示例编号:`[2026-07-14-01]`。

## 四、链接与事实核验(硬约束,违反即整条作废)

1. **禁止凭记忆编造 URL。** 每个链接必须出现在你当天真实的搜索结果中,且搜索返回的标题/摘要与你的声明一致。
2. 引用了具体数字(百分比、金额、倍数)的主张,尽量找**两个独立来源**交叉印证;只有单一来源时,在条目内标注"(单一来源)"。
3. 若你的执行环境无法直接抓取页面全文(如学术域名被防火墙拦截),用搜索摘要核验并在当日文件头注明"链接经搜索摘要核验,未逐页打开全文"。
4. 拿不准的数字,宁可降为定性表述("显著更高"),不要把宽区间讲死。
5. **诚实是唯一标准**:检索不到就不写;每天产出末尾附一段「来源可靠性说明」,列出哪些条目你自己觉得偏弱。

## 五、每日执行流程

1. **恢复上下文**:
   - 读 `lessons/` 目录全部文件(若存在)——里面是历史踩坑经验(鸡汤来源特征、高质量来源清单、审计教训)。
   - 读 `research/daily/DAILY_INDEX.md`(若存在)——看最近 7 天已收录的主题,今天避开。
2. **研究**:选 1-2 个方向,真实联网检索,产出 5-8 条候选。
3. **质量门槛过滤**:逐条对照第一节四条门槛 + 各方向红线,不达标直接丢弃(在日志中记录丢弃原因)。目标净产出 **3-5 条**;真货不足 3 条就只交 2 条,并在日志说明。
4. **去重(硬约束)**:
   - 与 `research/INSIGHTS.md`(主库 109 条)逐一比对;
   - 与 `research/daily/` 下最近 30 天的条目比对;
   - 同义条目丢弃或合并(保留来源更硬的版本)。判断标准:两条的"怎么用"如果指向同一个动作,就是重复,哪怕包装不同。
5. **落盘**:
   - 新建 `research/daily/YYYY-MM-DD.md`,写入当日条目 + 来源可靠性说明;
   - 更新(或创建)`research/daily/DAILY_INDEX.md`:追加一行 `| 日期 | 方向 | 净入库数 | 丢弃数 | 当日最佳条目一句话 |`;
   - 有新经验(发现好来源、坏来源、判断技巧)→ 写入 `lessons/`(一条经验一个文件,已有同类文件就更新)。
6. **自审后再收尾**:报告前把每个声明对照当天真实的工具结果核对——条目数用命令实际统计(`grep -c '^### ' research/daily/YYYY-MM-DD.md`),不许估算。
7. **提交**(若在 git 仓库中运行):`git add research/daily lessons && git commit -m "daily insights YYYY-MM-DD: +N 条"` 并推送到当前分支。
8. **收尾汇报格式**(第一句必须是):`本日新增 X 条(方向:_),丢弃 Y 条,库内无重复。` 细节放后面。

## 六、给调度者的使用说明(人类阅读,agent 可忽略)

- **喂给 Codex/Hermes**:把本文件全文作为任务 prompt(或让 agent 先读本文件再执行),配合每日定时触发(cron / 平台自带的 scheduled task)。
- **前置条件**:agent 需要联网搜索能力 + 本仓库读写权限。若 agent 无法访问本仓库,把 `research/INSIGHTS.md` 的标题列表(`grep '^### ' research/INSIGHTS.md`)一并附在 prompt 里供去重。
- **产出去向**:`research/daily/` 每天一个文件,`DAILY_INDEX.md` 是速览表。周末可人工把当周最佳条目提升进主库 `research/INSIGHTS.md` 对应领域。
- **质量抽查建议**:每周随机抽 2-3 条,换关键词自己搜一遍核验数字——本任务的价值完全建立在"来源硬"上。
