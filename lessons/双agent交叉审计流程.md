# 双 agent 交叉审计流程(本任务最有效的质量保障)

## 做法
- 每个领域入库后,派一个**独立的第二遍审计 agent**,明确要求"用与原研究员不同的搜索词重新检索、交叉验证核心数字",而不是重复点原链接。
- 审计 agent 输出三级判定:✅ PASS / ⚠️ MINOR(写清偏差+修正建议)/ ❌ FAIL(写清证据)。
- 审计与新领域研究**并行**跑(每轮 2 研究 + 1 审计),不串行等待。

## 为什么有效(本任务实测)
- 6 轮共审计 9 个领域,累计 101 PASS / 12 MINOR / 0 FAIL。**12 处 MINOR 全部是"方向对但数字/口径有小偏差"**,例如:
  - 薪资加薪获批率 15%→16%
  - 金钱贫困率 16.1%/9.1% 改为可核实的 13.2%/8.8%、"85+ 女性占 81%"→"六成以上"
  - 千禧一代财务让渡 51%→61%
  - 健身脂肪供能精确百分比无法独立复现 → 降为定性 + 补运动员边界条件
  - ACL 降险"25-33%"与"OR≈0.4"打架 → 统一为"OR 0.4-0.8/降险 20-60%"
  - 法律-10 的 181 号案例裁判要点、生育-11 HFEA 已改五档、安全-09 FBI 数据年份、皮肤-10 妊娠分级已废止
- 这些偏差**第一遍研究 agent 都没抓到**,是第二个 agent 换搜索词才发现的。单遍研究会把它们当"已验证"带进最终库。

## 关键细节
- 审计 agent 也会诚实报告"具体数字未能独立复现"(如健身 50.9/43.7、心理 50-75% 漏诊率)——这类要么降为定性表述,要么标注"待复检",不硬报通过。
- 出网策略屏蔽学术域名直接抓取时,审计只能靠 WebSearch 摘要;要在最终交付里如实标注"链接经摘要核验、未逐页打开全文"。
- 搜索额度可能在长循环中耗尽(皮肤 03-13 的第二遍就撞上了);撞上时让 agent 明确区分"已独立重搜"vs"仅领域知识评估",不含糊。
