## 二、第一笔"被动收入"路线图(按顺序做)

### 第 1 步:免费上线(只剩一次点击 ✅ 其余已由 Claude 完成)

已完成的部分:代码已合并到 `main`,自动部署已跑通,游戏成品已发布到 `gh-pages` 分支。

**你只需要做一件事**(GitHub 不允许机器人代做这一步):

1. 打开 <https://github.com/Skyrocktor124/cosmos-discovery-game/settings/pages>
2. 在 **Build and deployment → Source** 选 **Deploy from a branch**
3. Branch 选 **`gh-pages`**,目录选 **`/ (root)`**,点 **Save**
4. 等 1-2 分钟,你的游戏就上线了:
   `https://skyrocktor124.github.io/cosmos-discovery-game/`

以后每次更新代码合并到 `main`,都会自动重新发布,不用再碰设置。
这一步不赚钱,但它给了你一个**可以发给任何人的链接**——后面所有变现都建立在这上面。

### 第 2 步:上架 itch.io(30 分钟,可以开始收钱)

[itch.io](https://itch.io) 是全球最大的独立游戏平台,上架免费、审核宽松:

1. 注册账号 → Dashboard → Create new project
2. 类型选 **HTML**,把 `npm run build` 生成的 `dist` 文件夹打成 zip 上传(勾选 "This file will be played in the browser")
3. 定价选 **"$0 or donate"**(免费+打赏)—— 免费游戏才有人玩,打赏是第一笔收入
4. 写好介绍、截几张游戏截图、打上标签:`space`, `procedural-generation`, `exploration`, `idle`

itch.io 会给你带来自然流量(平台自己有浏览用户),打赏通过 PayPal 直接到账。

### 第 3 步:提交到网页游戏聚合平台(广告分成,真正的被动收入)

这些平台**替你带流量、替你放广告、按月给你分成**,这是网页小游戏被动收入的主流模式:

| 平台 | 模式 | 说明 |
|------|------|------|
| [CrazyGames](https://developer.crazygames.com) | 广告分成 ~50% | 月活上亿,提交后审核,通过就躺赚 |
| [Poki](https://developers.poki.com) | 广告分成 | 门槛稍高,但流量质量最好 |
| [GameMonetize](https://gamemonetize.com) | 广告分成 | 门槛低,适合练手 |
| [itch.io](https://itch.io) | 打赏/付费 | 第 2 步已覆盖 |

提交方式基本都是:填表 + 给游戏链接。如果平台要求接入他们的广告 SDK(比如 CrazyGames 要求在加载/过场时机调用广告接口),**把要求发给我,我来写代码**。

### 第 4 步(可选,做大之后):自有域名 + Google AdSense

等游戏每天有几百个访问后:

1. 买个域名(约 ¥60/年,唯一的成本)
2. 域名指向 GitHub Pages(Settings → Pages → Custom domain)
3. 申请 [Google AdSense](https://adsense.google.com),把广告代码给我,我嵌到页面里

## 三、对收入的诚实预期

| 阶段 | 现实的数字 |
|------|-----------|
| 刚上线,没推广 | ¥0(没有流量就没有收入,这是物理定律) |
| itch.io + 2-3 个聚合平台,3 个月 | 每月几十到几百元 |
| 某个平台推荐位/小爆款 | 每月几百到几千元 |
| 多个游戏组成矩阵(见下) | 收入 × 游戏数量 |

**被动收入的真正玩法是"矩阵"**:一个小游戏月入 200 元不起眼,但 10 个就是 2000 元,
而且每个都是做完就不用管。你可以持续让我帮你产出新游戏,复用今天搭好的这套
"零成本 + 自动部署 + 多平台上架"流水线。
