# /event-plan — 活动执行

先读 `training-coordinator/centre-profile.md`。与用户讨论用中文，执行计划用英文。

## 流程

1. **确定形式和基础信息**：读当前活动文件夹的 `03-tor.md` 和 `04-speakers.md`；确认线上/线下/混合（ToR 里通常已定）。线下活动还要问：城市/场地状态、预计人数、有无餐饮和 networking 环节。

2. **产出 `05-event-plan.md`** 写入当前活动文件夹，包含：

   **A. Countdown checklist**（从活动日反推为真实日期，责任人一栏留空由用户填）：
   - T–5/6w：ToR 定稿、讲者邀请发出、平台/场地预订
   - T–4w：讲者确认、议程锁定、注册页文案与上线
   - T–3w：宣传启动（Centre newsletter、LinkedIn、合作伙伴渠道）、speaker briefing 发出
   - T–2w：报名进度检查、二轮宣传、收集讲者 slides 初稿
   - T–1w：slides 定稿审阅、prep call/彩排、主持词定稿（提示运行 /script）、reminder email
   - T–1d：最终名单导出、当天 reminder、tech check 安排确认
   - T+0：run sheet 执行
   - T+1w 内：录像/slides 分发、感谢邮件、问卷结果汇总、**把实际讲者加入 speakers-exclusion-list.md**、内部复盘

   **B. Day-of run sheet**（精确到分钟的当天流程表）：
   - 线上版：开播前 30 分钟起 — tech check、开录、放行观众、各环节切换、Q&A 管理分工（谁盯 Q&A box）、结束后立即事项
   - 线下版：场地布置、签到、物料（roll-up、名牌、handouts、问卷二维码）、餐饮/茶歇时点、设备（麦克风、翻页器、备用笔记本）、拍照、networking 安排

   **C. Roles & responsibilities**：moderator、tech host、Q&A 管理、注册台/后台、宣传对接各一行

   **D. Risk & contingency**：讲者缺席、平台故障/设备故障、报名不足（含补救宣传方案）、超时处理

   **E. Promotion plan**：渠道清单、发布时点、注册目标及检查点

3. 高亮需要用户立即行动的头 3 件事（按今天日期和倒排时间线判断哪些已经紧迫）。

## 注意

- 倒排日期一律换算成真实日历日期，遇周末顺延到工作日。
- 混合形式活动把 A/B 两套要点合并，不要分成两份文档。
