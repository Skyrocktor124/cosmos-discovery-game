## -1. 世界观（先读这个）

1. **百万爆款是概率事件，不是配方产物。** 任何承诺"照做必爆"的公式都是假的。
   系统能做的是三件事：提高单发命中率（时效×情绪×工艺）、提高发射次数（周更3条）、
   在信号出现时立刻加注（某条起势就趁热追拍续集）。成熟账号的百万条大约也就
   每20-50条出一条。**评价系统看6条以上的趋势，不看单条。**
2. **脚本只占一条视频成败的一部分。** 画面、声音、字幕节奏、封面、发布时间、账号权重
   都在分母里。脚本之外的变量不进日志（第10节尸检协议），迭代就是盲改。
3. **每条只改一个主变量。** Alisa(爆)→翁荔(扑)一次换了三个变量（人物档位、资产类型、时效），
   等于什么都没学到。换主角是最贵、信息量最低的改法，放在最后才用。

## 0. 账号模型（一句话）

**做中英互联网的信息差搬运工：英文互联网这周炸了什么华人/女性/逆袭故事，
在中文观众还不知道的时候，第一个用漂亮的英文讲给他们听，并留下可收藏资产。**

Alisa 条爆的真正结构是：她的博客在英文圈刚刚病毒式传播 4 天 → 中文观众完全不知道 →
你是第一个搬进抖音的人 → 观众转发的是"你听说了吗"（新闻冲动），不是"给你介绍个厉害的人"。
翁荔/Oakley/Lucy Guo 三条全是"维基百科条目"：故事是旧的，中文媒体早已铺过
（Lucy Guo 的富豪新闻 2025 年中文圈就刷屏过），没有信息差 = 没有"第一个告诉你"的传播冲动。
- 账号：七克英语成长（英语学习+成长类，粉丝画像：英语学习者、关注教育的家长、职场女性）
- 双重价值：故事情绪价值 + 英文学习素材价值（地道短句可跟读→反复观看+收藏）
- 已验证系列：《打开黑箱的人 / They Opened the Black Box》
- 系列签名金句：**"Transparency is the new leverage."**（每条结尾用其变体收尾，形成追更预期）

## 1. 选题公式（v2.0 修正）

**爆款人物 = 时效（第一乘数）× 反差身份 × 打开黑箱 × 可收藏资产**

### ⛔ 保鲜期铁律（v2.0，凌驾于其他所有选题规则）
- **人物条必须挂在一个 ≤7-14 天的新鲜事件上**（TA刚发的爆款帖/刚拿的奖/刚上的热搜/刚开源的项目）。
  挂不上的人物一律降级为备用素材，不进主档期。
- 立项前必答两个问题：①"这件事发生几天了？" ②"中文互联网已经报过了吗？"
  事件>14天 或 中文圈已刷屏 → 信息差为零 → 不做主打条。
- 选题来源从"想一个厉害的人"改为"**扫描英文互联网过去一周炸了什么**"（/baokuan radar）：
  X/Twitter 病毒帖、Hacker News 首页、GitHub Trending、Substack 爆文、突发奖项/离职/发布。
  在其中筛华人/女性/逆袭 + 免费资产的故事。

- 反差身份三件套：华人 / 女性 / 博士（或学生、普通人）。每个标签一个情绪按钮：
  华人→民族自豪（高转发）；女性→女性认同（配 #女性智慧，高转发+高认同评论）；
  博士/学生→知识崇拜或"我也可以"代入感。
- 打开黑箱：面试黑箱、薪资黑箱、名校课程黑箱、前沿研究黑箱。黑箱=不公感=张力。
- 可收藏资产（本账号的命脉）：免费、公开、可搜索的具体东西。
  立项前必答："观众存下这条视频，是为了以后回来找什么？"答不出=不立项。

### ⛔ 铁律：资产必须是"本账号观众"用得上的（v1.1，翁荔条血的教训）
**"客观上很牛"≠"我的观众想存"。** 翁荔咖位比 Alisa 高，但她的资产（讲扩散模型/PPO的研究博客）
对英语学习者、家长、职场女性零用途 → 收藏引擎熄火 → 扑街。立项时用观众画像过一遍：
- 资产是否是**这群人**（英语学习者/成长追求者/职场女性/家长）明天就能用上的？
- 主角是**逆袭者**还是**已在山顶的人**？逆袭张力=点赞和转发的来源，山顶的人几乎没有张力。
- 判断顺序：**先问资产对不对得上观众，再看人物咖位。** 咖位是加分项，不是入场券。
- 本账号最安全的资产类型：能直接用的成长/求职/学习方法（如面试地图、谈薪脚本、自学路线），
  外加口播本身的地道英文（可跟读=二次收藏）。最危险：需要专业门槛才能消费的前沿研究。
- 人物节奏：女性人物与"普通人"人物交替排期，喂养两个已验证情绪按钮
  （女性认同 + "我也可以"），防止系列变成大神展览。

## 2. 九拍口播结构（已验证）

| 拍 | 任务 | 要点 |
|---|---|---|
| 1 | 数字悬念开场 | 3-4个**精确**数字连砸（57比"几十"强一个量级），主语用 She/He，**不报名字** |
| 2 | 反常识动作 | 一个违反直觉的行为 + 2-3词短句加压（"For free."单独成句） |
| 3 | 身份揭晓 | "This is ___." 名字压到第6句左右；身份标签逐句放出，每个标签单独成句 |
| 4 | 细节堆叠 | 列举5-8项具体资产内容，专业细节要具体到外行觉得"哇"、内行觉得"真"。具体性=可信度 |
| 5 | 升维转折 | 固定转场句 "Now here is why this story matters." + 黑箱隐喻，把个人故事拉成行业议题 |
| 6 | 群体指认 | 两个排比 "Especially if you are ___." 直接点名目标观众——这是转发按钮 |
| 7 | 隐喻升华 | 贯穿隐喻的顶点（黑箱→地图、门→钥匙），主人公的选择用两个短句收（"She drew it. And she handed it to everyone."式） |
| 8 | 具体的人着陆 | 落到"somewhere, a ___ read it and thought: I can actually do this."——大道理不让人流泪，具体的人才让人流泪。决定共鸣型评论 |
| 9 | 金句收尾+评论钩子 | "X is the new Y"式反常识金句（可截图可转述）+ **二选一提问**（"换作是你，你会公开吗？"）——评论是当前短板，必须给出口 |

### 语言检查清单（逐条过）
- [ ] 每句 ≤ 12 个英文单词，一句一个信息点（口播节奏=字幕节奏）
- [ ] 至少3组排比（"Not X. Not Y. Just Z." / "All public. All free." / "Especially… Especially…"）
- [ ] 对抗结构贯穿全篇（隐藏vs公开、黑箱vs地图）
- [ ] 第一个词就是信息，零"大家好/今天讲"
- [ ] 每个数字、头衔、事件都经过核实，无出处的删掉
- [ ] 金句密度足够（喂收藏）；英文地道、可跟读（喂英语学习者的收藏和复看）

### 片长与再钩（v1.0 新增，来自首条复盘）
- 目标片长 60-90 秒；若超过，**每20秒左右加一个再钩**：
  把最炸的细节拆开预告（"她公开的东西里，有一份连大厂员工都在传——这个后面讲"）。
- 首条视频流失点在中后段（节拍4之后），中段必须持续给"再看10秒的理由"。

## 2.4 定位关卡：高价值干货博主（v2.1，凌驾于叙事技巧）

**本账号是干货博主，不是故事博主。故事只是干货的证据，不是主菜。**
爆款条（Alisa）本质是干货密度最高的一条：一句一个硬货（PPO vs GRPO、谈薪脚本、offer时间线），
51%收藏率由此而来。v2.0 之前的翻车稿共同点：一条视频只讲一个道理，情绪浓、知识稀=口水化。

硬指标：
1. **干货密度 ≥ 每15秒一条可带走的知识**（观众关掉视频还能复述并使用的东西）。
   自检法：把稿子里"能写进笔记的句子"划出来，少于4条=打回重写。
2. **每条干货必须"具体到能立刻用"**：说"股权比工资值钱"不够，要给出她留3%变10亿的实证+
   一句可执行的结论。抽象口号=口水。
3. **编号清单体**是与9拍叙事体并列的第二种已验证结构：
   新闻钩子(带时间戳) → "我提炼了N条，做好笔记" → Rule 1…N(每条=金句+实证+可执行结论) →
   显式收藏CTA("Save this") → 数字投票评论钩("哪条最狠？1-5")。
   干货主打条优先用清单体；人物情绪条才用9拍体。
4. **显式收藏CTA**：干货条结尾必须有"Save this / 存下来慢慢用"——收藏是本账号命脉，
   干货条不喊收藏等于白给。

## 2.5 硬性关卡（v2.0，写稿不过关不出稿）

1. 片长双轨制（2026-07-11 用户定版）：**主打干货条走 3 分钟中视频路线**（爆款条 Alisa 即约3分钟，
   实战样本优先于60秒推演）。3分钟版的代价条款：干货密度 ≥ 每13-15秒一条可带走知识（全片≥12条）、
   **每45秒一个再钩**（预告后面更狠的一条）、封面必须埋中段悬念（"第X条最狠"）拉完播。
   60秒短版保留为分众切片/破圈条的格式。
2. **每稿交 3 个开头钩子**（前3句的三个不同版本：数字悬念版 / 冲突提问版 / 新闻播报版），
   标注推荐项。开头决定 5 秒完播，5 秒完播决定第一个流量池，这是最值得 A/B 的位置。
3. **新闻感开场词**：主打条开头必须带时间戳词（"Three days ago…" / "This week…" / "就在昨天"），
   把"新"写在第一句里——新闻感本身就是钩子。
4. **每条只改一个主变量**并在 data-log 里登记本条的实验变量是什么（见第9节）。

## 3. 标题 / 标签 / 封面

- 中文标题句式："一个XX的XX，改写了XX的规则" / "她做了一件事，整个XX圈都沉默了" /
  "没人想到，XX圈最狠的人是个XX"。要点：不点名、身份标签前置、动作动词要大。
- 标签三件套：情绪标签（#女性智慧 #华人之光）+ 热点标签（#AI）+ 垂直标签（账号定位相关）。
- 封面文案：中文，突出身份反差+大动作，与标题互补不重复。

## 4. 发布与运营检查清单

- [ ] 置顶评论：教观众**用什么关键词搜索**资产（不放站外链接，抖音严打引流）
- [ ] 发布后1小时内，作者亲自回复前排评论5-10条（作者回复会被顶上去=免费高赞评论位；
      首条验证：作者回复拿60赞）
- [ ] 结尾评论钩子已埋（二选一提问）
- [ ] 预埋2-3条备用回复：回应"怎么找到TA"、回应情绪共鸣型评论
- [ ] 不使用"被限流/被忌惮"叙事（评论区自然出现是助力，作者亲自说是风险）
- [ ] 爆款出现后2-4周内连发同系列（人群标签沉淀窗口期）

## 5. 数据诊断基准（滚动修正）

| 指标 | 首条实测（Alisa Liu条） | 健康线 | 诊断含义 |
|---|---|---|---|
| 5秒完播 | 40.43% | ≥40% 及格，≥50% 优秀 | 开场钩子强度 |
| 完播率 | 2.57%（短板） | 长视频≥10%起步 | 中后段留存；决定流量池晋级，当前最大杠杆 |
| 收藏/点赞 | 51%（极高，账号命脉） | 本账号≥30% | 资产价值成立与否的检验指标 |
| 转发/点赞 | 8% | ≥5% | 情绪按钮（群体指认拍）是否命中 |
| 评论/点赞 | 1.3%（短板） | ≥3% | 评论钩子与互动运营 |
| 推荐页占比 | 98% | ≥80%为破圈 | 是否走出粉丝盘 |

诊断顺序：先看收藏/点赞比（模型是否成立）→ 再看完播（天花板在哪）→
再看评论/转发（哪个情绪出口堵了）→ 最后读高赞评论（观众未满足需求=续集选题）。

## 6. 已验证结论（只增不删，标注来源条目）

1. 【条目#1】收藏驱动是本账号的爆款引擎：收藏/点赞达51%，观众把视频当"资源入口"。
   → 每条必须埋可收藏资产。
2. 【条目#1】极端收藏+转发数据可以在低完播下硬拉进推荐池（98%推荐页），
   但完播率(2.57%)是天花板。→ 完播是当前第一优化目标。
3. 【条目#1】"两个Especially"群体指认拍有效：高赞评论出现"女人就是这样，磊落…"（161赞）。
4. 【条目#1】评论区的资源型提问（"怎么搜到TA"41赞）= 现成续集选题 + 置顶评论必答项。
5. 【条目#1】"搜不到/稀缺感"会推高收藏，但阴谋论叙事只能让它在评论区自然发生。
6. 【条目#1】作者下场回复有效（60赞），前排回复是零成本数据放大器。
7. 【条目#2｜翁荔扑街】资产对不上观众=收藏引擎熄火=扑街。impressiveness ≠ saveability。
   选人先过观众画像，再看咖位。前沿研究类人物（李飞飞除外，因其资产是"改变世界"的
   国民级叙事）暂时不做——太远。
8. 【条目#2】逆袭张力是点赞/转发的来源。"已在山顶的人"缺张力，即使资产可收藏也难爆。
   下一条要回到 Alisa 式的逆袭者。
9. 【推论】评论区被验证的需求 > 任何新选题。479+41赞在要 Alisa 的地图 →
   最高命中率的一条是"交付被反复索要的资产"，而非硬找新人。ride the proven winner。

## 8. 破圈模块（v1.2）——画像过窄时，加宽入口而非重定位

**背景**：某条数据画像"92%高知年轻女性、圈层偏窄"，撞到播放瓶颈。
**总原则**：破圈=多修上匝道，不是换高速。保住核心引擎（华人/女性/逆袭情绪 + 地道英文素材 +
可收藏干货），只在"入口"加宽。**过度泛化杀两样命根：①精确细节带来的可信度与收藏；②英文护城河。**

1. **情绪锚点降维（保具体，换共鸣）**：把行业黑话换成人人可对标的成长命题（"改写AI圈规则"→
   "普通女孩靠3个选择拿到offer"）——但**保留精确数字与硬细节做可信度**。
   反例警戒：别把"57场面试"抹成"面试了很多次"，那等于自杀式泛化。
2. **泛标签叠加（不删精准标签）**：核心 #女性智慧 #英语学习 保留1-2个，另加2-3个泛流量标签
   #普通人成长 #职场干货 #女生必看。老粉匹配度不掉，新流量池打开。
3. **场景多入口（分众裂变）**：同一人物/主题裂变成分众切片，各自自带"可用小技巧"=各自的收藏钩子——
   学生（考研复试/社团面试）、30+转行女性、少量男性向（男生面试踩坑）。一鱼多吃，覆盖不同年龄与性别。
4. **单条可独立传播的小技巧**：从大故事里切一个脱离视频也能用的干货点（"HR问缺点怎么答"），
   门槛低、转发高、能被非本圈用户直接用。
5. **转发钩子**：结尾加低门槛社交指令（"转给正在准备面试的闺蜜"），撬社交链做跨圈传播，
   与已有的二选一评论钩子并存（一个催评论、一个催转发）。

**刹车提醒**：破圈动作会稀释初始互动速度（算法晋级的关键来自核心粉丝的高互动密度）。
所以核心情绪与英文素材永远不动，泛化只加在"入口层"（标签/场景/小技巧/转发语），
不动"内核层"（人物选择/情绪结构/英文品质）。

## 9. 实验纪律（v2.0）

- 排期表同时是**实验计划表**：每条登记"本条相对上一条改了什么唯一主变量"
  （开头钩子型 / 片长 / 人物档位 / 时效性 / 发布时间 / 画面形式…）。
- 一条扑了 ≠ 推翻模型。先做尸检（第10节）定位是选题层、脚本层还是制作层的问题，
  再决定下一条改哪个变量。**连续两条在同一变量上失败，才修改该变量的规则。**
- 起势信号出现（发布后2小时点赞增速异常）→ 立刻准备续集，48小时内追发，趁人群标签热。

## 10. 尸检协议（v2.0）——不交上一条的尸检，不写下一条

每条视频发布满48小时必须登记 data-log，缺一项就问用户要一项：

**数据层**：播放量、5秒完播、完播率、点赞/评论/转发/收藏、推荐页占比、粉丝画像变化
**制作层**（脚本之外的成败变量，v1.x 一直失明的部分）：
- 实际片长；配音方式（真人男/女声、AI声、语速）；画面形式（图文轮播/实拍/素材混剪）；
- 字幕样式（中英双语？关键词卡点？）；封面图与封面文案实际用了什么；
- 发布时间（星期几+几点）；发布时账号近7天更新频率
**判定**：5秒完播低→开头或封面问题；5秒高完播低→中段节奏或片长问题；
完播尚可但播放卡在几千→选题时效/账号权重问题；数据全好但没破圈→标签与破圈层问题。
